# ナレッジ記事テンプレート（Markdown・既定）

`<…>` を差し替えて使う。流れは「遭遇した問題 → 原因 → 解決」。設計知見系なら「遭遇した問題」を「このドキュメントの射程」等に置き換えてよい。

```markdown
---
title: <簡潔なタイトル>
tags: [<tech>, <topic>, ...]
---

## TL;DR

- 結論を箇条書きで先に。

## 遭遇した問題

- 何が起きたか。再現条件・環境（バージョン等）を残す。

## 原因

- なぜそうなるか。

## 解決

- どう直したか / どう設計したか。

```ts
// 最小再現に絞ったコード。動かない例と動く例を併記すると後で役立つ。
```

## まとめ

- 一行まとめ。要点を短く。

## 参考

- 参考リンク・出典。
```
